## 第二阶段：API接口清单

### 认证接口 (AuthController - `/api/v1/auth`)
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/login` | 用户登录 |
| POST | `/refresh` | 刷新令牌 |
| POST | `/logout` | 用户登出 |

### 用户接口 (UserController - `/api/v1`)
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/users` | 创建用户 |
| GET | `/users` | 获取用户列表 |
| GET | `/users/{user_id}` | 获取用户详情 |
| PUT | `/users/{user_id}` | 更新用户 |
| DELETE | `/users/{user_id}` | 删除用户 |
| POST | `/users/change-password` | 修改密码（需认证） |
| GET | `/me` | 获取当前用户信息（需认证） |

### RBAC接口 (RBACController - `/api/v1/rbac`)
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/roles` | 创建角色 |
| GET | `/roles` | 获取角色列表 |
| GET | `/roles/{role_id}` | 获取角色详情 |
| PUT | `/roles/{role_id}` | 更新角色 |
| DELETE | `/roles/{role_id}` | 删除角色 |
| GET | `/permissions` | 获取权限列表 |
| POST | `/permissions/init` | 初始化系统权限 |
| POST | `/users/{user_id}/roles` | 分配角色给用户 |
| DELETE | `/users/{user_id}/roles/{role_id}` | 移除用户角色 |
| GET | `/users/{user_id}/roles` | 获取用户角色 |
| GET | `/users/{user_id}/permissions/check` | 检查用户权限 |

### 安全接口 (SecurityController - `/api/v1`)
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/security/blacklist` | 添加IP黑名单 |
| GET | `/security/blacklist` | 获取黑名单列表 |
| DELETE | `/security/blacklist/{ip}` | 移除黑名单 |
| POST | `/security/whitelist` | 添加IP白名单 |
| GET | `/security/whitelist` | 获取白名单列表 |
| DELETE | `/security/whitelist/{ip}` | 移除白名单 |
| POST | `/security/rate-limit` | 创建限流规则 |
| GET | `/security/rate-limit` | 获取限流规则列表 |
| PUT | `/security/rate-limit/{rule_id}/toggle` | 切换限流规则 |
| DELETE | `/security/rate-limit/{rule_id}` | 删除限流规则 |
| GET | `/security/status` | 获取安全状态 |

### 系统管理接口 (SystemController - `/api/v1/system`)
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/health` | 健康检查 |
| POST | `/depts` | 创建部门 |
| GET | `/depts` | 获取部门列表 |
| GET | `/depts/tree` | 获取部门树 |
| GET | `/depts/{dept_id}` | 获取部门详情 |
| PUT | `/depts/{dept_id}` | 更新部门 |
| DELETE | `/depts/{dept_id}` | 删除部门 |
| POST | `/menus` | 创建菜单 |
| GET | `/menus` | 获取菜单列表 |
| GET | `/menus/tree` | 获取菜单树 |
| GET | `/menus/{menu_id}` | 获取菜单详情 |
| PUT | `/menus/{menu_id}` | 更新菜单 |
| DELETE | `/menus/{menu_id}` | 删除菜单 |
| POST | `/roles` | 创建角色 |
| GET | `/roles` | 获取角色列表 |
| GET | `/roles/{role_id}` | 获取角色详情 |
| PUT | `/roles/{role_id}` | 更新角色 |
| DELETE | `/roles/{role_id}` | 删除角色 |
| POST | `/roles/{role_id}/menus` | 分配菜单权限 |
| GET | `/roles/{role_id}/menus` | 获取角色菜单 |
| POST | `/users/{user_id}/roles` | 分配用户角色 |
| GET | `/users/{user_id}/roles` | 获取用户角色 |
| GET | `/users/{user_id}/menus` | 获取用户菜单 |
| GET | `/operation-logs` | 获取操作日志列表 |
| GET | `/operation-logs/{log_id}` | 获取日志详情 |

## 执行步骤

1. **移动数据库文件**
2. **修改 development.py 配置**
3. **运行迁移验证** - `python manage.py migrate`
4. **编写单元测试** - 覆盖所有接口
5. **运行单元测试** - `pytest tests/`
6. **启动服务** - `python manage.py runserver`
7. **运行集成测试** - `python scripts/integration_test.py`
